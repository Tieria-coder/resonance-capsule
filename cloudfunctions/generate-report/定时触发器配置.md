# 定时触发器配置说明

## 推送时间设置

| 报告类型 | 推送时间 | Cron表达式 | 参数 |
|:---|:---|:---|:---|
| 日报 | 每天晚上23:00 | `0 0 23 * * * *` | `type=daily` |
| 周报 | 每周日晚上23:00 | `0 0 23 ? * SUN *` | `type=weekly` |
| 月报 | 每月最后一天晚上23:00 | `0 0 23 L * ? *` | `type=monthly` |

## 云开发控制台配置步骤

### 1. 打开云开发控制台
微信开发者工具 → 云开发 → 云函数 → generate-report

### 2. 创建定时触发器

#### 日报触发器
- 触发器名称：`daily-report-trigger`
- 触发周期：`自定义触发周期`
- Cron表达式：`0 0 23 * * * *`
- 时区：`Asia/Shanghai`
- 触发消息：`{"type":"daily"}`

#### 周报触发器
- 触发器名称：`weekly-report-trigger`
- 触发周期：`自定义触发周期`
- Cron表达式：`0 0 23 ? * SUN *`
- 时区：`Asia/Shanghai`
- 触发消息：`{"type":"weekly"}`

#### 月报触发器
- 触发器名称：`monthly-report-trigger`
- 触发周期：`自定义触发周期`
- Cron表达式：`0 0 23 L * ? *`
- 时区：`Asia/Shanghai`
- 触发消息：`{"type":"monthly"}`

### 3. 部署云函数
修改代码后，右键 `generate-report` → 上传并部署：云端安装依赖

## Cron 表达式说明

格式：`秒 分 时 日 月 星期 年`

```
┌───────────── 秒 (0-59)
│ ┌───────────── 分 (0-59)
│ │ ┌───────────── 时 (0-23)
│ │ │ ┌───────────── 日 (1-31，L表示最后一天)
│ │ │ │ ┌───────────── 月 (1-12)
│ │ │ │ │ ┌───────────── 星期 (SUN-SAT 或 0-6，?表示不指定)
│ │ │ │ │ │ ┌───────────── 年 (可选)
│ │ │ │ │ │ │
* * * * * ? *
```

**特殊字符：**
- `*` 任意值
- `?` 不指定（日和星期互斥，一个用?另一个用具体值）
- `L` 最后一天（用于日字段）
- `SUN` 星期日

## 示例

```
0 0 9 * * * *      # 每天早上9点
0 0 23 * * * *     # 每天晚上11点
0 0 23 ? * SUN *   # 每周日晚上11点
0 0 23 L * ? *     # 每月最后一天晚上11点
0 30 8 1 * ? *     # 每月1号早上8:30
```

## 注意事项

1. 定时触发器在云开发控制台配置，不在代码中
2. 修改云函数代码后需要重新部署
3. `package.json` 中不需要配置定时触发器
4. 如需测试，可在云函数详情页手动触发测试
