# 情绪细分系统改造 - 2026-05-22 14:50

## 目标
将原有的 7 个扁平情绪改为「8大类 × 3-4子情绪」的二级结构，覆盖全面且边界清晰。

## 最终情绪架构
8大类 × 3-4子情绪 = 约25个子情绪：

| 大类 | 子情绪 | 颜色 |
|:---|:---|:---|
| 😊 开心 | 愉悦/满足/感恩 | #4CAF50 |
| 😌 平静 | 放松/释然/充实 | #8BC34A |
| 🤩 兴奋 | 激动/期待/惊喜 | #FF9800 |
| 😢 难过 | 失落/沮丧/孤独/遗憾 | #5C6BC0 |
| 😰 焦虑 | 担心/着急/心慌 | #FF5722 |
| 😠 生气 | 烦躁/委屈/愤懑/无奈 | #F44336 |
| 😴 疲惫 | 疲劳/无力/犯困 | #795548 |
| 😳 羞愧 | 尴尬/自责/懊悔 | #9C27B0 |

## 修改文件清单

| 文件 | 改动 |
|:---|:---|
| `utils/util.js` | 重构 EMOTIONS → EMOTION_CATEGORIES（8类+子情绪）；`getEmotionByKey()` 支持子情绪 key 解析 |
| `pages/index/index.js` | data 新增 `emotionCategories/selectedCategory/tempSubEmotion/showSubEmotions`；新增 `selectCategory/selectSubEmotion/backToCategories` 方法；`confirmRecord` 改用子情绪 key |
| `pages/index/index.wxml` | 情绪选择改为两步：第一步选大类 → 第二步选子情绪 |
| `pages/index/index.wxss` | 新增 `.category-grid/category-option/sub-emotion-grid/sub-emotion-option` 样式 |
| `pages/report/report.js` | `EMOTION_COLORS` 配色表更新为8大类配色 |
| `pages/calendar/calendar.js` | `getEmotionByKey` 返回值适配子情绪；日历图标/标签显示「开心·愉悦」格式 |

## 选择流程（用户视角）
1. 点击记录 → 弹窗显示 8 个大类图标
2. 点击一个大类 → 下方展开 3-4 个子情绪选项（含描述）
3. 点击子情绪 → 选中后显示「开心·愉悦」
4. 底部文字输入 → 提交保存

## 数据库兼容
- 旧数据（情绪 key 如 `excited`、`happy`）仍能通过 `getEmotionByKey` 正常解析
- 新数据存储子情绪 key（如 `happy_joy`）
- 报告/日历等展示层统一解析为「大类·子情绪」格式
